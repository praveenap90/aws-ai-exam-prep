▸ Exam Blueprint

The 5 Exam Domains

The AIF-C01 exam covers five weighted domains. Understanding each domain's weight helps you focus your study time effectively.

DOMAIN 01

Fundamentals of AI & ML

⚖ 20% of exam

Core concepts of AI, ML, and deep learning. Types of ML (supervised, unsupervised, reinforcement), model training, evaluation, and the ML pipeline.

Supervised Learning Neural Networks Model Evaluation Bias & Variance Feature Engineering

DOMAIN 02

Fundamentals of Generative AI

⚖ 24% of exam

Foundation models, large language models, prompt engineering, tokens, embeddings, and the lifecycle of generative AI applications.

Foundation Models Prompt Engineering Tokens & Embeddings RAG Fine-tuning

DOMAIN 03

Applications of Foundation Models

⚖ 28% of exam

Design patterns for GenAI applications, selecting the right model, Amazon Bedrock capabilities, agents, knowledge bases, and deployment patterns.

Amazon Bedrock Agents for Bedrock Knowledge Bases Model Selection Guardrails

DOMAIN 04

Guidelines for Responsible AI

⚖ 14% of exam

AI ethics, bias mitigation, fairness, transparency, privacy, and AWS tools for responsible AI development and deployment.

AI Bias Fairness Explainability AWS AI Ethics Privacy

DOMAIN 05

Security, Compliance & Governance

⚖ 14% of exam

Securing AI workloads on AWS, compliance frameworks, data governance, IAM for AI services, and encryption for AI data.

IAM for AI Encryption VPC Endpoints Compliance Audit Logging

▸ Quick Reference

Cheat Sheet

Key concepts and mnemonics to memorize before exam day.

🔢 ML Types at a Glance

  • Supervised → labeled data, predict output
  • Unsupervised → unlabeled data, find patterns
  • Reinforcement → rewards/penalties, learn behavior
  • Self-supervised → generates own labels (LLMs)

📐 Model Evaluation Metrics

  • Accuracy → (TP+TN) / total
  • Precision → TP / (TP + FP)
  • Recall → TP / (TP + FN)
  • F1 Score → harmonic mean of P & R
  • AUC-ROC → discrimination ability
  • RMSE → regression error magnitude

🧠 GenAI Key Terms

  • Token → smallest unit of text (≈ ¾ word)
  • Embedding → numerical vector of meaning
  • Context window → max tokens model can handle
  • Temperature → controls output randomness
  • Hallucination → confident but wrong output
  • RAG → ground FM with your own data

⚡ Prompt Engineering Techniques

  • Zero-shot → no examples, direct question
  • Few-shot → 2-5 examples in prompt
  • Chain-of-thought → "think step by step"
  • System prompt → sets context/persona
  • ReAct → reason + act in loops

🛡️ Responsible AI Pillars

  • Fairness → unbiased across groups
  • Explainability → why did it decide X?
  • Privacy → protect PII/PHI
  • Robustness → stable under adversarial input
  • Transparency → honest about AI usage
  • Veracity → accurate, grounded outputs

🔐 Security Must-Knows

  • Bedrock uses VPC endpoints (no public internet)
  • IAM roles → least privilege for AI services
  • CloudTrail → audit logs for Bedrock API calls
  • AWS Macie → PII detection in S3 datasets
  • KMS → encrypt model artifacts & data

🏗️ Bedrock Quick Facts

  • No ML expertise needed to use Bedrock
  • Data NOT used to train base models
  • Models: Claude, Llama, Mistral, Titan, etc.
  • Fine-tuning → custom model on your data
  • Continued pre-training → domain adaptation
  • Model evaluation → compare FM outputs

📊 SageMaker Key Features

  • Studio → unified ML IDE
  • Autopilot → AutoML for tabular data
  • Clarify → bias detection + SHAP explainability
  • Pipelines → MLOps CI/CD for ML
  • Model Monitor → detect data drift in prod
  • Ground Truth → managed data labeling
