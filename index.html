▸ Quick Reference

Cheat Sheet

Last-minute scan reference — formulas, mnemonics, and service lookup tables. For deeper study and self-testing, use the Flash Cards ↗ section.

📐 Model Evaluation Metrics — Formulas

  • Accuracy → (TP+TN) / total predictions
  • Precision → TP / (TP + FP) — minimise false alarms
  • Recall → TP / (TP + FN) — minimise missed cases
  • F1 Score → 2 × (P × R) / (P + R) — use for imbalanced data
  • AUC-ROC → model's discrimination ability (0.5 = random)
  • RMSE → regression error magnitude (lower = better)
  • ROUGE → summarization quality (recall-focused)
  • BLEU → translation quality (precision-focused)

⚡ Prompt Engineering — Quick Mnemonics

  • Zero-shot → 0 examples, direct ask
  • One-shot → 1 example to show format
  • Few-shot → 2–5 examples for consistency
  • Chain-of-thought → "think step by step"
  • Role prompt → "You are an expert in..."
  • System prompt → sets persona for whole session
  • RAG prompt → inject retrieved docs as context
  • Stop sequence → defines where output ends

🔐 AI/ML Security — Exam-Day Context

  • Bedrock → VPC Endpoints keep traffic off public internet
  • SageMaker → IAM Role (not user keys) grants S3 access
  • Training data → Macie scans S3 for PII before training
  • Model artifacts → KMS encrypts at rest
  • API calls → CloudTrail audits every Bedrock invocation
  • Guardrails → apply content filters across ALL Bedrock models
  • Prompt injection → mitigated by Guardrails + input validation

🎯 Bias–Variance — Quick Decision

  • High bias + Low variance → Underfitting → simplify assumptions
  • Low bias + High variance → Overfitting → regularize, add data
  • High bias + High variance → Rare, unstable → redesign model
  • Low bias + Low variance → Good Fit ✅ → maintain & monitor
  • Fix underfitting → more features, increase complexity
  • Fix overfitting → L1/L2 regularization, dropout, more data

🤖 Amazon Bedrock

What is Amazon Bedrock?

A fully managed service that gives you access to high-performing foundation models (FMs) from leading AI companies via a single API. No infrastructure to manage, no ML expertise required.

Key Capabilities

Model Access — Claude (Anthropic), Llama (Meta), Titan (Amazon), Mistral, Stable Diffusion, Cohere via one API.
Knowledge Bases — Managed RAG: index your documents and ground FM responses in your data.
Agents — Autonomous multi-step workflows using FMs + APIs + Lambda.
Fine-tuning — Adapt a FM with your labeled data. Updates model weights.
Continued Pre-training — Domain adaptation using large unlabeled text corpora.
Guardrails — Safety filters: content blocking, PII redaction, topic denial.
Model Evaluation — Compare FM outputs with automated or human review.

Pricing Model

Pay per input + output token. No upfront cost, no minimum spend.
Provisioned Throughput available for predictable, high-volume workloads.

GENERATIVE AI · KNOWLEDGE BASES

📚 Bedrock Knowledge Bases

What are Bedrock Knowledge Bases?

A fully managed RAG (Retrieval-Augmented Generation) implementation. Connects your FM to your own documents and data sources so it can answer questions grounded in your content — without retraining.

How it Works

1. Ingest — Upload documents to S3 (PDFs, Word, HTML, etc.).
2. Chunk — Documents are split into chunks automatically.
3. Embed — Each chunk is converted to a vector embedding (using Titan Embeddings or Cohere Embed).
4. Store — Embeddings stored in a vector DB (OpenSearch, Aurora, Pinecone, Redis).
5. Retrieve — At inference, user query is embedded and matched to relevant chunks.

🕵️ Bedrock Agents

What are Bedrock Agents?

Agents autonomously plan and execute multi-step tasks by combining a foundation model with tools (APIs, Lambda functions, data sources). Users give instructions in natural language; the agent figures out how to complete the task.

Agent Components

Foundation Model — The reasoning engine (e.g. Claude). Decides what actions to take.
Instructions — System prompt that defines agent purpose and behavior.
Action Groups — APIs or Lambda functions the agent can call (e.g. get_order_status, send_email).
Knowledge Base — Optional: agent can retrieve information from documents via RAG.

Agent Execution Flow

1. User sends natural language request → 2. Agent creates plan → 3. Calls action groups/APIs → 4. Queries knowledge base if needed → 5. Synthesizes final response.

Fundamentals of AI & ML

Types of Machine Learning

Supervised — Labeled data, predict output. Examples: classification, regression.
Unsupervised — Unlabeled data, find patterns. Examples: clustering, anomaly detection.
Reinforcement — Agent learns via reward/penalty. Examples: game AI, robotics.
Semi-supervised — Small labeled + large unlabeled data combined.
Self-supervised — Model generates its own labels from structure in data (how LLMs are trained).

ML Lifecycle Steps

1. Define business problem → 2. Collect & prepare data → 3. Feature engineering → 4. Train model → 5. Evaluate → 6. Deploy → 7. Monitor & retrain

Overfitting vs Underfitting

Overfitting — Great on training data, poor on test data. Fix: more data, dropout, regularization.
Underfitting — Poor on both training and test. Fix: more complex model, more features.

Evaluation Metrics

Accuracy — (TP+TN)/total. Misleading for imbalanced datasets.
Precision — TP/(TP+FP). Use when false positives are costly (e.g. spam filter).
Recall — TP/(TP+FN). Use when false negatives are costly (e.g. cancer detection).
F1 Score — Harmonic mean of Precision & Recall. Best for imbalanced classes.
RMSE — Root Mean Square Error. Used for regression tasks.
AUC-ROC — Model's ability to distinguish between classes.

Fundamentals of Generative AI

Core GenAI Terms

Token — Smallest unit of text an LLM processes (~¾ of a word). Pricing is per token.
Context window — Maximum tokens the model can process at once (input + output combined).
Temperature — Controls randomness. 0 = deterministic/predictable. 1 = creative/random.
Top-P (nucleus sampling) — Considers tokens whose cumulative probability ≥ P. Narrows creative range.
Embedding — Numerical vector that represents the semantic meaning of text. Used in RAG.
Hallucination — Model confidently generates factually incorrect information.

Prompt Engineering Techniques

Zero-shot — No examples given. "Translate to French: Hello."
One-shot — Exactly one example provided before the task.
Few-shot — 2–5 examples in the prompt to demonstrate desired format.
Chain-of-thought — "Think step by step." Forces model to reason before answering.
System prompt — Sets overall persona, rules, and context for the model session.
ReAct — Reason + Act in loops. Used in agents.

RAG (Retrieval-Augmented Generation)

User query → retrieve relevant docs from knowledge base → inject into prompt → model generates grounded answer.
Reduces hallucinations. No model retraining needed. Data stays up to date.

Tokenization

Tokenization is the process of breaking text into smaller units (tokens) before sending it to the AI model. Pricing for LLMs is based on token count.

Text: "AWS makes AI easy" → Tokens: AWS | makes | AI | easy
On average, 1 token ≈ ¾ of a word. 100 tokens ≈ 75 words.
Context window = max tokens the model can process at once (input + output combined).

Embeddings

Embeddings convert text into numerical vectors that AI systems can understand — capturing semantic meaning so similar concepts have similar vectors.

Embeddings are used for:

Semantic search — Find documents by meaning, not just keyword match
Similarity detection — Compare how related two pieces of text are
Recommendation systems — Surface similar items to a user
RAG retrieval — Find the most relevant document chunks to inject into a prompt

IAM Policy Types

Understanding the five policy types is essential for designing secure, compliant AWS environments.

Policy TypeAttached ToPurposeExample Use CaseKey Feature
Identity-Based Policy IAM Users, Groups, Roles Grants permissions to identities to access AWS resources Allow a data scientist to run training jobs in Amazon SageMaker Most commonly used
Resource-Based Policy AWS resources (S3 bucket, Lambda) Specifies who can access a resource Allow another account to read data from an Amazon S3 bucket Policy stored directly on the resource
Permissions Boundary IAM Users or Roles Defines the maximum permissions an identity can have Limit a developer role so it cannot create admin policies Acts as a permission ceiling
Service Control Policy (SCP) AWS Accounts within AWS Organizations Restricts permissions across multiple AWS accounts Prevent all accounts from launching certain services Works at organization level

🎯Identity-Based = most common. Attached to the IAM user/role. Controls what they CAN do.

🎯Permissions Boundary = sets the CEILING on what a role can ever be granted, regardless of identity policies.
⚡SCP restricts accounts in AWS Organizations. It does NOT grant permissions — it only limits them.
